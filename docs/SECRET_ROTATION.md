# Secret Rotation Guide — O.D.I.N.

How to rotate the two critical secrets in a deployed ODIN instance.

## When to Rotate

- Suspected compromise of encryption key or JWT secret
- Personnel with key access leaves the team
- Routine rotation (recommended: annually)

## Prerequisites

- SSH/shell access to the ODIN host
- Docker CLI available
- Downtime window (rotation requires a container restart)

---

## 1. Rotate JWT_SECRET_KEY

JWT secrets sign auth tokens. Rotating invalidates all active sessions — users must re-login.

```bash
# 1. Stop the container
docker compose down

# 2. Generate a new JWT secret
NEW_JWT=$(python3 -c "import secrets; print(secrets.token_bytes(32).hex())")

# 3. Write it to the persistent data volume
echo "$NEW_JWT" > /path/to/odin-data/.jwt_secret
chmod 600 /path/to/odin-data/.jwt_secret

# 4. Start the container — it reads the new key on boot
docker compose up -d
```

**Impact:** All active sessions and tokens are invalidated. Users must log in again. No data loss.

---

## 2. Rotate ENCRYPTION_KEY

The encryption key (Fernet) protects sensitive fields in the database:
- MFA TOTP secrets (`users.mfa_secret`)
- OIDC client secrets (`oidc_config.client_secret_encrypted`)
- SMTP passwords
- OIDC access tokens

Rotating requires re-encrypting all encrypted fields with the new key.

```bash
# 1. Stop the container
docker compose down

# 2. Save the old key (you'll need it for re-encryption)
OLD_KEY=$(cat /path/to/odin-data/.encryption_key)

# 3. Generate a new encryption key
NEW_KEY=$(python3 -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())")

# 4. Run the re-encryption script
docker compose run --rm \
  -e OLD_ENCRYPTION_KEY="$OLD_KEY" \
  -e ENCRYPTION_KEY="$NEW_KEY" \
  odin python3 -c "
from cryptography.fernet import Fernet
import sqlite3, os

old_fernet = Fernet(os.environ['OLD_ENCRYPTION_KEY'].encode())
new_fernet = Fernet(os.environ['ENCRYPTION_KEY'].encode())

db = sqlite3.connect('/data/odin.db')

# Re-encrypt MFA secrets
rows = db.execute('SELECT id, mfa_secret FROM users WHERE mfa_secret IS NOT NULL').fetchall()
for uid, encrypted in rows:
    if encrypted and encrypted.startswith('gAAAAA'):
        plaintext = old_fernet.decrypt(encrypted.encode()).decode()
        new_encrypted = new_fernet.encrypt(plaintext.encode()).decode()
        db.execute('UPDATE users SET mfa_secret = ? WHERE id = ?', (new_encrypted, uid))

# Re-encrypt OIDC client secrets
rows = db.execute('SELECT id, client_secret_encrypted FROM oidc_config WHERE client_secret_encrypted IS NOT NULL').fetchall()
for oid, encrypted in rows:
    if encrypted and encrypted.startswith('gAAAAA'):
        plaintext = old_fernet.decrypt(encrypted.encode()).decode()
        new_encrypted = new_fernet.encrypt(plaintext.encode()).decode()
        db.execute('UPDATE oidc_config SET client_secret_encrypted = ? WHERE id = ?', (new_encrypted, oid))

db.commit()
db.close()
print('Re-encryption complete.')
"

# 5. Write the new key
echo "$NEW_KEY" > /path/to/odin-data/.encryption_key
chmod 600 /path/to/odin-data/.encryption_key

# 6. Start the container
docker compose up -d
```

**Impact:** Brief downtime during re-encryption. No data loss. All encrypted fields are migrated to the new key.

---

## 3. Verify After Rotation

```bash
# Check container is healthy
docker compose ps

# Check logs for startup errors
docker compose logs --tail=20

# Test login (JWT rotation invalidates sessions)
curl -X POST http://localhost:8000/api/v1/auth/login \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "username=admin&password=yourpassword"

# If MFA is enabled, verify TOTP still works after encryption key rotation
```

## File Locations

| Secret | Env Var | Persistent File | Generated By |
|--------|---------|-----------------|--------------|
| JWT signing key | `JWT_SECRET_KEY` | `/data/.jwt_secret` | `docker/entrypoint.sh` |
| Encryption key | `ENCRYPTION_KEY` | `/data/.encryption_key` | `docker/entrypoint.sh` |
| Installation ID | — | `/data/.odin-install-id` | `docker/entrypoint.sh` |

All files are created with `chmod 600` (owner read/write only).
